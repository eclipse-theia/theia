## Vision

ArtixCode IDE aims to become a **Bangladesh-born AI coding environment** for developers, students, startups, and software teams.

The goal is not only to create another code editor. The goal is to build an intelligent development platform that can:

* understand a full codebase,
* debug real project issues,
* compare implementation with documentation,
* audit Laravel and SaaS projects,
* detect risky code patterns,
* generate safe patches,
* run tests and builds,
* create pull requests,
* and let developers approve changes before merging.

ArtixCode follows one core principle:

> **AI may propose. Tools may verify. Developers decide.**

## What is ArtixCode IDE?

**ArtixCode IDE** is a planned cloud and desktop development environment for AI-assisted software engineering.

It is designed to combine:

* a modern IDE interface,
* codebase indexing,
* AI chat and agent workflows,
* terminal and tool execution,
* Git-aware patching,
* safe workspace isolation,
* Laravel-focused audit intelligence,
* Bangla-English developer support,
* and future ArtixCode model families.

ArtixCode IDE is planned to support both:

1. **Cloud IDE**
   A browser-based development environment where users can open, run, scan, debug, and patch projects in isolated cloud workspaces.

2. **Desktop IDE**
   A local developer environment powered by Theia or similar open-source IDE foundations, with deep local project access.

## Cloud Coding Plan

ArtixCode Cloud Coding is planned as a browser-based development environment where users can connect a repository and work inside an isolated cloud workspace.

Planned workflow:

1. User connects a GitHub/GitLab repository.
2. ArtixCode creates a cloud workspace.
3. The project is cloned into an isolated container.
4. ArtixCode detects the tech stack.
5. The user can code, run commands, scan the project, debug errors, or request patches.
6. AI-generated changes are made in a safe branch/workspace.
7. Tests and builds are run in the workspace.
8. The developer reviews the diff.
9. ArtixCode creates a pull request if approved.

Cloud workspace support is planned for:

* Laravel,
* PHP,
* Composer,
* Node.js,
* React,
* Next.js,
* MySQL/PostgreSQL,
* Redis,
* queues,
* tests,
* build pipelines,
* and audit tools.

## Safe Patch Lab

**ArtixCode Patch Lab** is a planned safe-fix workflow.

Instead of modifying the developer’s original project directly, ArtixCode will create an isolated workspace.

Preferred Git workflow:

```bash
git worktree add ../artixcode-fix-workspace -b artixcode/fix-checkout-bug
```

Patch Lab workflow:

1. Original project remains untouched.
2. ArtixCode creates a branch or Git worktree.
3. AI applies fixes only inside the isolated workspace.
4. Tests/build/lint/security checks run there.
5. ArtixCode generates a report.
6. Developer reviews the diff.
7. Developer approves, rejects, exports patch, or creates a PR.

High-risk areas such as payment, authentication, authorization, order lifecycle, stock handling, and database migrations must require explicit human approval.

## Security Principles

ArtixCode IDE must be designed with security first.

Principles:

* Never directly modify the original project.
* Use isolated workspaces for AI-generated fixes.
* Run unknown code in sandboxed containers.
* Require permission before running commands.
* Do not expose secrets.
* Detect secrets but mask values.
* Do not train on private code without explicit consent.
* Require human approval for high-risk patches.
* Never auto-merge payment, authentication, authorization, order, stock, or security-sensitive changes.
* Use defensive security workflows only.
* Support audit logs for all agent actions.
